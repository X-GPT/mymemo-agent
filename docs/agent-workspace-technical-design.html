MyMemo agent workspace technical design
E2B runtime · OpenRouter gateway · durable workspace

Persistent workspace. Ephemeral sandbox. OpenRouter-backed gateway.

The target system gives users durable agent continuity while keeping compute warm only while useful, routing LLM calls through OpenRouter for cheaper models, and keeping documents, provider keys, and database access behind the unified gateway.

Core Decision

Use E2B for execution and OpenRouter through the gateway for LLMs. The sandbox is a hydrated working copy for untrusted agent execution. The control plane and gateway own state, tokens, documents, OpenRouter credentials, audit, model policy, and VPC database access.
Runtime Warm-on-demand Lease or create a sandbox for active work, sync state out, then terminate after a short idle window.
Boundary No direct DB access The sandbox never receives DATABASE_URL, AWS credentials, OPENROUTER_API_KEY, or direct provider keys.

System Shape

chat-api

Owns conversations, runs, workspace sync, sandbox leases, token minting, and SSE replay.

→

E2B sandbox

Runs daemon, agent, Bash/browser tools, local hydrated docs, and generated files during a lease.

↔

workspace store

Durable artifacts, workdirs, document manifests, snapshots, and run event logs.

unified gateway

OpenRouter-backed LLM proxy, document search/fetch, token enforcement, budgets, and audit.

Reachable only from the gateway security group. Never from E2B.

↔

OpenRouter

Gateway-only upstream for cheaper model routing and fallback policy.

Current Behavior

  • chat-api creates a fresh E2B sandbox per turn.
  • daemon.js and agent.js are deployed into that sandbox.
  • The sandbox agent calls the unified gateway for LLM and documents.
  • The gateway proxies LLM traffic to Anthropic today (ANTHROPIC_API_KEY, UPSTREAM_BASE_URL defaults to api.anthropic.com).
  • Target state: the gateway sends LLM traffic to OpenRouter by default.
  • Documents are not materialized to local disk by default.
  • mymemo-docs exposes separate search and fetch.
  • Claude SDK sessionId is an internal agentSessionId.

Sandbox Lifecycle

1. Request

Create a runId for the conversation.

2. Lease

Reuse a warm sandbox or create a new E2B sandbox.

3. Hydrate

Restore conversation workdir, manifests, and selected artifacts.

4. Execute

Run the agent and stream durable events.

5. Sync

Persist changed files, document manifest, outputs, and audit.

6. Recycle

Terminate after idle, version drift, health failure, or risk signal.

search_documents Contract

Internal Behavior

  • Search remote document index through the unified gateway.
  • Rank candidates.
  • Fetch selected top documents through the gateway.
  • Write fetched documents to local sandbox disk.
  • Update the conversation docs manifest.
  • Return snippets plus local file paths.

Agent Policy

  • Local docs are the working set, not the whole corpus.
  • Search remote by default for open document questions.
  • Use local-only only when the user explicitly says current or already-loaded files.

Gateway Policy

  • Separate llm and documents audiences.
  • Enforce user, conversation, run, scope, expiry, and limits.
  • Audit every search and hydrated document.

Sandbox Policy

  • No database credentials, OPENROUTER_API_KEY, or provider keys.
  • No full corpus copied by default.
  • Sandbox is disposable after state sync.

Unified Gateway Responsibilities

LLM Module

  • Verify bearer token.
  • Enforce audience and expiry.
  • Enforce model/header allowlists.
  • Inject OPENROUTER_API_KEY only on upstream requests.
  • Proxy sandbox LLM traffic to OpenRouter.
  • Select cheaper/default models by gateway policy, run config, or environment.
  • Preserve the sandbox-facing Claude Agent SDK compatibility surface where practical.

Document Module

  • Verify bearer token.
  • Enforce user, conversation, run, scope, and expiry.
  • Search private document DB.
  • Fetch scoped document content.
  • Enforce hydration limits.
  • Audit every search and hydrated document.
Sandbox must never receive DATABASE_URL, DB password, AWS credentials, OPENROUTER_API_KEY, direct provider API keys, or broad document credentials.

OpenRouter LLM Routing

Sandbox-Facing Env

ANTHROPIC_BASE_URL=<MYMEMO_GATEWAY_URL>
ANTHROPIC_AUTH_TOKEN=<short-lived llm_token>
ANTHROPIC_API_KEY=""

The sandbox still talks only to the MyMemo gateway. ANTHROPIC_AUTH_TOKEN is the MyMemo run token, not the OpenRouter API key.

Gateway Upstream Env

LLM_PROVIDER=openrouter
OPENROUTER_API_KEY=...
OPENROUTER_BASE_URL=https://openrouter.ai/api
OPENROUTER_DEFAULT_MODEL=anthropic/claude-sonnet-4.5
OPENROUTER_CHEAP_MODEL=google/gemini-flash-1.5
OPENROUTER_HTTP_REFERER=https://mymemo.example
OPENROUTER_APP_TITLE=MyMemo
Routing policy Provider selection is centralized in the gateway. The sandbox can request a capability or model class, but the gateway owns the allowlist, fallback rules, budget controls, and final OpenRouter model selection.

Audience Enforcement

  • Use aud = "llm" for LLM proxy requests.
  • Use aud = "documents" for document operations.
  • Prefer separate env vars: ANTHROPIC_AUTH_TOKEN and MYMEMO_DOC_TOKEN.
  • ANTHROPIC_AUTH_TOKEN is the MyMemo short-lived gateway token, not OPENROUTER_API_KEY.
  • Even in one deployed gateway, each route family rejects the wrong audience.

Major Gaps From Current Code

Runtime And State

  • Current code creates and kills a fresh sandbox per turn.
  • No sandbox lease manager or warm idle policy exists.
  • No durable workspace store or sync manifest exists.
  • No conversationId or runId exists.
  • Claude SDK sessionId is exposed as sessionId; treat it as agentSessionId.

Documents And Tokens

  • mymemo-docs exposes separate search and fetch.
  • The prompt instructs manual search then fetch.
  • Documents are not hydrated to local disk as first-class local paths.
  • Tokens do not include aud, conversationId, or runId.
  • Gateway needs an OpenRouter provider adapter and model policy for cheaper-model routing.
  • Gateway is one deployable service but should be split internally.

Non-Goals

  • Do not give the sandbox direct database access.
  • Do not copy the full user corpus into the sandbox by default.
  • Do not treat local docs as the complete corpus.
  • Do not use Claude SDK sessionId as conversationId or runId.
  • Do not expose OPENROUTER_API_KEY to the sandbox, daemon child agent, client, or document tools.
  • Do not make Fargate the default runtime unless AWS-native control becomes more important than E2B's agent sandbox abstraction.
